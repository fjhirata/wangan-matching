## 1. 今回、何を変えたか（ひとことで）

**「性格診断でタワマンの“棟”を提案」から、「診断＋条件で “いま実際に売り出し中の部屋” を提案」へ**進化させました。
（＝ポータルサイトのように、その場で買える具体的な住戸が出るようになりました）

### 主な変更点
| 項目 | 変更前 | 変更後 |
|---|---|---|
| 提案するもの | 建物（棟）のおすすめ | **いま募集中の住戸**（約1,072戸）からTOP3 |
| 最初の入力 | 年収・年齢 | **予算・広さ・間取り・築年数・駅徒歩**（実際に探す条件） |
| 診断 | 5項目・20問・13タイプ | **6項目（眺望を追加）・24問・15タイプ** |
| 結果の見せ方 | 棟の紹介 | 住戸TOP3＋**建物をタップで“その棟の募集中の部屋”を一覧**／条件を変えてその場で再検索 |
| プライバシー配慮 | ― | 階数・面積は「30階台・70㎡台」のように**少しぼかして表示**（部屋の特定を防止） |

## 2. データの取り方（募集DBとの連携）

### どこから取っているか
参照元は **「★湾岸マンション募集DB ver2」（Googleスプレッドシート）** です。以下を自動で読み取っています（**読み取り専用**。シートは一切書き換えません）。

- **募集マスタ（エリア別タブ）** … いま売り出し中の住戸データ（価格・広さ・階数・間取り・方位）。
  毎月末の記録列のうち **最新（現在は2026/5/31）** に価格がある部屋を「現在募集中」として抽出 → **約1,072戸**。
- **成約履歴タブ** … 過去に成約した価格。ここから各建物の「相場（坪単価）」を計算。
- **物件スペック（別シート）** … 駅徒歩・築年・耐震・総戸数・共用施設。物件名で結合（49棟中48棟をカバー）。

### 更新のされ方
- スプレッドシートが更新されれば、**毎週（月曜朝）自動でアプリに反映**されます。
- 今表示されているのは **2026/5/31 時点** のデータです。

## 3. 点数（スコア）の付け方・ロジック

利用者の「価値観診断の結果」と、各物件の「実力」を突き合わせて**マッチ度**を出しています。

### ① 物件を6つの観点で0〜100点に採点（すべて実データから算出）
| 観点 | 何を根拠に点数化しているか |
|---|---|
| 資産性 | 過去成約の**値上がり率** |
| 広さ | 募集住戸の**面積** |
| 施設 | **共用施設**（プール・ジム・ラウンジ等）の充実度 |
| 立地 | **駅徒歩**＋エリア |
| 居住性 | **築年・耐震・設備**（内廊下・オール電化など） |
| 眺望 | **階数 ÷ その建物の総階数**（高層ほど高得点）※現状は階数ベース。将来、方角（海/運河向き）等も追加可能 |

### ② マッチ度の出し方
- 利用者が**重視する観点ほど重み**を大きくし、物件の点数と掛け合わせてマッチ度（％）を算出。
- 診断で「資産性重視」と出た人には資産性の高い物件が、「眺望重視」の人には高層階が上位に来ます。

### ③ 坪単価の出し方（実データと推計を区別）
- **成約実績がある建物** … 実際の成約坪単価の平均（12棟）＝「成約」と表示
- **成約実績がない建物** … いまの募集価格の中央値（37棟）＝「募集」と表示
- どちらを使っているか**必ず画面に明記**（実データと推計を混同させない方針）

### ④ 絞り込み（必須条件）
- 予算・広さ・間取り・築年数・駅徒歩は**必須条件**として扱い、**合致する募集中の住戸だけ**を表示。
- 条件に合う部屋が無ければ「**該当なし**」と正直に表示します。

## 4. 補足・現状の制約（正直な注記）

- 階数・面積は**ぼかし表示**（部屋の特定を避けるため）。正確な情報はLINE相談へ誘導。
- 新築1棟（THE TOYOMI TOWER）のみスペック未連携のため一部項目が非表示。
- 各スコア・坪単価は**実データに基づく“算出値”**です（不動産会社としての分析価値を担保しつつ、実測と推計を明示）。
- 現状は**プロトタイプ／社内確認用のプレビュー**です。

## 5. さわり方（確認方法）
1. 上記URLを開く → 「無料で診断をはじめる」
2. 条件（予算・広さ・間取り 等）を選ぶ → 24問の診断に回答
3. 結果画面で「いま狙えるTOP3」と価値観チャートを確認
4. 各物件カードをタップ → その建物の募集中の部屋一覧
5. 画面下部で条件を変えて「この条件で再検索」もその場で可能
